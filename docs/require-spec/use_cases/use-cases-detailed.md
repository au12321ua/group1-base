# 详细用例设计（第2步：模板化说明）

## 范围说明
- 本文档基于基础用例进行详细展开。
- 主要参与者分为两类：普通用户、管理员用户。
- 用例编号与基础用例保持一致。

## UC-01 账号登录
- 用例名：账号登录
- 主要参与者：普通用户
- 目标：通过账号密码完成身份认证并进入系统。
- 前提条件：用户账号存在且处于可用状态；用户已获取登录入口。
- 触发器：用户在登录页点击“登录”。
- 场景：
	1. 用户在登录页输入账号和密码并提交。
	2. 系统校验凭据合法性并签发访问凭证。
	3. 用户进入个人首页并建立会话状态。
	4. 凭证临近过期时，系统通过续期机制维持会话连续性。
- 异常：账号不存在；密码错误；账号被禁用；凭证签发失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 鉴权模块负责凭据校验与token签发。
	2. 网关负责后续请求中的token校验与身份透传。
- 未解决的问题：是否要求普通用户首次登录后必须修改初始密码。

## UC-02 查看个人信息
- 用例名：查看个人信息
- 主要参与者：普通用户
- 目标：查看本人档案信息。
- 前提条件：用户已登录；用户身份已通过鉴权。
- 触发器：用户进入个人中心并点击“查看资料”。
- 场景：
	1. 用户进入个人中心并发起资料查看请求。
	2. 系统读取该用户档案信息并返回结果。
	3. 页面按规则展示字段：基础身份字段只读，可编辑字段独立展示。
- 异常：用户会话过期；用户档案不存在；数据服务异常。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块提供本人档案查询接口。
	2. 网关透传用户身份字段。
- 未解决的问题：是否允许教师/学生查看部分公共字段（如院系）以支持业务协作。

## UC-03 修改个人可编辑信息
- 用例名：修改个人可编辑信息
- 主要参与者：普通用户
- 目标：更新联系方式、邮箱、紧急联系人等可编辑字段。
- 前提条件：用户已登录；目标字段属于可编辑范围。
- 触发器：用户提交个人信息编辑表单。
- 场景：
	1. 用户修改可编辑字段并提交表单。
	2. 系统校验字段格式和字段级权限。
	3. 系统保存变更并返回最新资料。
	4. 系统记录变更时间和操作者信息。
- 异常：字段格式错误；尝试修改只读字段；并发更新冲突；保存失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块执行字段级校验与持久化。
	2. 审计日志模块记录变更轨迹。
- 未解决的问题：是否需要对敏感字段（手机号、邮箱）增加二次校验。

## UC-04 上传头像或个人照片
- 用例名：上传头像或个人照片
- 主要参与者：普通用户
- 目标：更新个人头像或照片。
- 前提条件：用户已登录；上传文件可访问。
- 触发器：用户在个人中心提交图片文件。
- 场景：
	1. 用户选择并上传头像或照片文件。
	2. 系统校验文件类型与大小。
	3. 系统存储文件并生成可访问地址。
	4. 系统更新用户档案中的图片地址。
- 异常：文件类型不支持；文件过大；存储失败；图片链接回写失败。
- 优先级：中
- 次要参与者及其使用方式：
	1. 文件存储服务负责保存图片文件。
	2. 信息管理模块负责维护图片元数据。
- 未解决的问题：图片审核是否需要人工流程。

## UC-05 退出登录
- 用例名：退出登录
- 主要参与者：普通用户
- 目标：结束当前会话，防止他人继续使用账户。
- 前提条件：用户已登录。
- 触发器：用户点击“退出登录”。
- 场景：
	1. 用户点击退出登录。
	2. 系统使当前会话或token失效。
	3. 客户端清理本地凭证并跳转到登录页。
- 异常：服务端失效操作失败；客户端本地清理失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 鉴权模块执行token失效处理。
	2. 前端应用执行本地凭证清理。
- 未解决的问题：是否支持“退出所有设备登录”。

## UC-06 修改密码
- 用例名：修改密码
- 主要参与者：普通用户
- 目标：将账户密码从旧密码更新为新密码。
- 前提条件：用户已登录；用户知道当前密码；新密码满足策略。
- 触发器：用户提交“旧密码+新密码”表单。
- 场景：
	1. 用户输入旧密码和新密码并提交。
	2. 系统校验旧密码正确性和新密码强度。
	3. 系统更新密码哈希并返回成功结果。
	4. 用户后续登录使用新密码。
- 异常：旧密码错误；新密码不符合策略；更新失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 鉴权模块负责密码校验与更新。
	2. 审计日志模块记录密码变更事件（不记录明文）。
- 未解决的问题：是否设置密码历史限制（禁止与最近N次相同）。

## UC-07 管理员登录
- 用例名：管理员登录
- 主要参与者：管理员用户
- 目标：以管理员身份登录并进入管理工作台。
- 前提条件：管理员账号有效且具备管理权限。
- 触发器：管理员在管理入口提交登录凭据。
- 场景：
	1. 管理员在管理入口输入凭据并提交。
	2. 系统按管理员策略执行更严格的身份校验。
	3. 系统签发管理员会话凭证并进入管理工作台。
- 异常：凭据错误；账号被禁用；账号无管理员角色；签发失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 鉴权模块执行高强度策略校验。
	2. 网关根据角色放行管理接口。
- 未解决的问题：是否要求管理员开启二次验证。

## UC-08 检索与查看用户信息
- 用例名：检索与查看用户信息
- 主要参与者：管理员用户
- 目标：按条件快速检索目标用户并查看详情。
- 前提条件：管理员已登录；管理员具有用户查询权限。
- 触发器：管理员输入检索条件并执行查询。
- 场景：
	1. 管理员输入检索条件并发起查询。
	2. 系统按学号/工号、姓名、角色等条件返回列表。
	3. 管理员点击目标记录查看详情。
- 异常：查询条件非法；分页超限；服务超时。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块提供筛选与分页查询能力。
	2. 日志模块记录查询操作。
- 未解决的问题：是否需要脱敏展示部分敏感字段。

## UC-09 新增用户
- 用例名：新增用户
- 主要参与者：管理员用户
- 目标：新增学生或教师账户并完成可登录准备。
- 前提条件：管理员已登录且具备新增权限；输入信息满足必填规则。
- 触发器：管理员提交新增用户表单。
- 场景：
	1. 管理员提交新增用户表单。
	2. 系统在信息管理板块创建用户基础档案并生成主键。
	3. 系统触发鉴权板块创建认证账号与初始凭据。
	4. 系统聚合处理结果并统一返回状态。
- 异常：档案创建失败；鉴权账号创建失败；跨模块调用超时；出现部分成功。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块作为主写方，负责建档。
	2. 鉴权模块消费创建请求并建立认证账号。
	3. 审计模块记录新增轨迹。
- 未解决的问题：跨模块一致性采用消息队列+重试，还是同步事务编排。

## UC-10 编辑用户与权限调整
- 用例名：编辑用户与权限调整
- 主要参与者：管理员用户
- 目标：更新用户信息并调整角色、启用状态等权限相关配置。
- 前提条件：管理员已登录；管理员具有编辑与授权权限。
- 触发器：管理员在详情页提交修改内容。
- 场景：
	1. 管理员提交用户信息或权限调整。
	2. 系统校验操作权限与字段有效性。
	3. 系统保存信息变更和角色调整。
	4. 新配置立即生效并作用于后续访问控制。
- 异常：越权调整；角色冲突；并发覆盖；保存失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块保存基础字段变更。
	2. 鉴权模块同步角色与账号状态。
	3. 审计模块记录高风险操作。
- 未解决的问题：角色调整是否需要审批流或双人复核。

## UC-11 批量导入用户（仅新增）
- 用例名：批量导入用户（仅新增）
- 主要参与者：管理员用户
- 目标：通过文件一次性新增多名用户并设置初始密码。
- 前提条件：管理员已登录；导入文件格式符合模板；系统已配置初始密码策略。
- 触发器：管理员上传CSV并点击“开始导入”。
- 场景：
	1. 管理员上传CSV并启动导入任务。
	2. 系统解析文件并逐条校验数据合法性。
	3. 系统仅对不存在用户执行新增建档。
	4. 系统触发鉴权账号创建并设置初始密码。
	5. 系统返回成功/失败统计与错误明细。
- 异常：文件格式错误；重复用户；部分记录导入失败；鉴权创建失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块处理批量建档。
	2. 鉴权模块批量创建认证账号并写入初始密码。
	3. 日志模块记录导入任务过程与结果。
- 未解决的问题：初始密码是文件直接提供，还是按统一规则自动生成并回传。

## UC-12 删除用户（逻辑删除）
- 用例名：删除用户（逻辑删除）
- 主要参与者：管理员用户
- 目标：将用户从常规业务可见集合移除，但保留恢复能力。
- 前提条件：管理员已登录且具有删除权限；目标用户存在且未被逻辑删除。
- 触发器：管理员在列表或详情页点击“删除”。
- 场景：
	1. 管理员发起删除操作。
	2. 系统将目标用户标记为逻辑删除并移入回收站。
	3. 常规查询结果中不再显示该用户。
	4. 系统同步使目标账户不可登录。
- 异常：目标已删除；删除权限不足；状态更新失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块维护逻辑删除状态。
	2. 鉴权模块同步账号禁用状态。
	3. 审计模块记录删除行为。
- 未解决的问题：逻辑删除后的数据保留期限如何定义。

## UC-13 回收站恢复用户
- 用例名：回收站恢复用户
- 主要参与者：管理员用户
- 目标：将逻辑删除用户恢复为可用状态。
- 前提条件：管理员已登录；目标用户位于回收站。
- 触发器：管理员在回收站点击“恢复”。
- 场景：
	1. 管理员在回收站选择目标用户并点击恢复。
	2. 系统恢复用户在信息管理板块中的可用状态。
	3. 系统将用户重新纳入常规列表展示。
	4. 必要时同步恢复鉴权登录状态。
- 异常：目标不存在；恢复冲突（同学号/工号冲突）；同步失败。
- 优先级：中
- 次要参与者及其使用方式：
	1. 信息管理模块执行用户状态恢复。
	2. 鉴权模块同步启用账号。
- 未解决的问题：恢复时若关键唯一字段已被占用，系统采用何种冲突处理策略。

## UC-14 回收站物理删除（支持批量）
- 用例名：回收站物理删除（支持批量）
- 主要参与者：管理员用户
- 目标：永久清理回收站中的用户数据。
- 前提条件：管理员已登录；目标记录处于逻辑删除状态。
- 触发器：管理员在回收站选择一条或多条记录并点击“物理删除”。
- 场景：
	1. 管理员在回收站选择一条或多条记录并提交物理删除。
	2. 系统执行不可恢复删除并清理用户档案数据。
	3. 系统清理认证关联数据。
	4. 系统返回批量处理结果。
- 异常：部分记录删除失败；关联数据清理失败；批量任务中断。
- 优先级：中
- 次要参与者及其使用方式：
	1. 信息管理模块删除主记录。
	2. 鉴权模块删除认证账户。
	3. 审计模块记录批量清理操作。
- 未解决的问题：物理删除是否需要二次确认和延迟生效窗口。

## UC-15 管理课程等基础信息
- 用例名：管理课程等基础信息
- 主要参与者：管理员用户
- 目标：维护课程及其他基础信息，供业务系统稳定使用。
- 前提条件：管理员已登录且拥有对应基础信息管理权限。
- 触发器：管理员在管理工作台对课程或其他基础对象发起增删改查操作。
- 场景：
	1. 管理员选择课程或其他基础信息对象并发起增删改查操作。
	2. 系统按规则校验输入字段和业务约束。
	3. 系统执行新增/查询/编辑/删除并返回结果。
	4. 变更结果对后续业务模块可见并可读取。
- 异常：字段校验失败；主键或编码冲突；删除受引用约束；保存失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块提供课程及基础信息API。
	2. 其他业务模块消费更新后的主数据。
- 未解决的问题：课程删除采用逻辑删除还是物理删除，及其对历史业务数据的影响。

## UC-16 访问受保护资源
- 用例名：访问受保护资源
- 主要参与者：普通用户、管理员用户
- 目标：在权限控制下访问允许的功能与数据。
- 前提条件：用户已登录并携带有效身份凭证。
- 触发器：用户访问需要鉴权的页面或接口。
- 场景：
	1. 用户携带身份凭证访问受保护页面或接口。
	2. 网关校验token并提取身份信息。
	3. 业务接口根据角色与权限点执行授权判断。
	4. 授权通过则返回数据，授权失败则返回无权限提示。
- 异常：token无效或过期；权限不足；身份信息透传失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 网关执行统一鉴权拦截。
	2. 鉴权模块提供token验证能力。
	3. 业务服务执行细粒度授权。
- 未解决的问题：细粒度权限采用RBAC，还是RBAC+资源级策略混合模型。

## UC-17 向排课业务组提供业务数据
- 用例名：向排课业务组提供业务数据
- 主要参与者：排课业务组（系统内部调用方）
- 目标：按排课业务口径获取教师名单、待选课名单、校历与选课学生名单。
- 前提条件：调用方通过服务态鉴权；具备相应数据访问权限。
- 触发器：排课业务组发起名单或校历查询请求。
- 场景：
	1. 调用方携带服务身份访问对应查询接口。
	2. 系统校验调用方权限并解析查询条件（学期、院系、专业、年级、课程班等）。
	3. 系统返回教师名单、待选课名单、校历或选课学生名单数据。
	4. 返回结果包含分页信息与snapshotTime/version。
	5. 调用方可按权限触发CSV导出。
- 异常：权限不足；查询条件非法；目标学期不存在；数据源不可用；导出任务失败。
- 优先级：高
- 次要参与者及其使用方式：
	1. 信息管理模块提供教师名单、待选课名单与校历查询能力。
	2. 选课学生名单由选课系统提供，本模块仅提供统一查询入口（不持久化记录）。
	3. 日志模块记录查询与导出行为。
- 未解决的问题：待选课名单口径是否严格按培养方案过滤已修课程与互斥课程。

## UC-18 向选课业务组提供培养方案
- 用例名：向选课业务组提供培养方案
- 主要参与者：选课业务组（系统内部调用方）
- 目标：按专业、年级、版本查询培养方案（含课程ID列表）。
- 前提条件：调用方通过服务态鉴权；已具备培养方案访问权限。
- 触发器：选课业务组发起培养方案查询请求。
- 场景：
	1. 调用方提交专业、年级、版本等查询条件。
	2. 系统校验条件并定位对应培养方案。
	3. 系统返回培养方案主体信息与requiredCourseIds列表。
	4. 返回结果包含version与snapshotTime，便于跨系统口径对齐。
